UI/UX: Follow the design process with steps to develop end-to-end products plus delivery, document reasonable decisions, and solve problems related to UX.

Competitive research

Before jumping to design a solution, I discover exist / relevant solutions to have an insight, learn from them, and then take first notes.

User research

I observed data from environment, did some academic researches, discussed with relevant people to create some pieces of data that's value.

Analyze data

I use an affinity mapping to organize data into themes. After that, analyze them to figure out critical points. I focus on all aspects of user from problems, behaviours to their need and motivation.

Create personas

I divide and combine data of aspects from the previous step into meaningful data prefer to personas. I separate all types of users into different categories and cut them down as specific as I could.

Sketch solutions

I apply Crazy 8's method to quickly generate first solutions. Then, I ask questions: “How may we” to optimize these solutions.

Choose solutions

I once again use an affinity mapping to organize solutions into themes. I discuss/research to locate them to a prioritization framework. Thanks for my front-end knowledge, it helps me to make decisions much easier.

Users journal

I create storyboards based on different types of user. I build a sitemap and then user flows by make senses and researches.

Prototype concepts

I use random an UI Kit from Figma to quickly create very first versions with incomplete / incorrect contents.

Document options

Decisions are made during steps of design process. I save matter decisions plus reasons.

Brand guidelines

I start with a message, missions and attributes to generate fonts, colors, layouts, and then complete Brand guidelines.

Develop patterns

I create components from atoms (colors, fonts, icons,...) with consideration for the implement and performance in coding.

Complete design

After implement and test components / systems design in codebase. I continue with Figma by bring all of them together to build high-fidelity mockups.

First version

I finalize the first version of design with interactions at basic. I can prototype by using Figma to illustrate User-flows. Or, coding by implement HTML / CSS / JavaScript to create Interfaces with simple features for testing.

Responsive designs

I consider making responsive designs to match with user behaviour. I cut down unnecessary features for simple experiences in small platforms.

Usability testing

I separate steps to accomplish the goal to create Usability Tasks. By implement Hotjar, I can record user behaviour at each step of the process to analyze, get insights, discover problems related to, and offer solutions.

Potential solutions

I list potential solutions along with the reason why I didn't choose plus conditions need to be had to apply for the development.

Design handoff

I use Zeplin to deliver all pieces of design to engineers (included use-cases).

** Coding (included in each step) **
